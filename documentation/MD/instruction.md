# MINI PROJECT GUIDELINES
**Due Date:** 05 October 2026, 23:59  
**Date of Notice:** 20-07-2026  

---

## 1. Project Requirements

*   **Group Size:** Maximum 3 students per group.
*   **Project Type:** Dynamic Web-Based Software Application.
*   **Minimum Functionalities:** 15 distinct functionalities.
*   **Minimum Web Pages:** 15 pages (excluding login, error, and utility pages).
*   **Database:** Relational database designed, created, and implemented by the students.
*   **Integration:** Database must be connected with the frontend via backend APIs.
*   **Technology Stack:** Open choice. Some examples include:
    *   **Frontend:** HTML5, CSS3, Bootstrap, JavaScript, React, Angular, Vue.js
    *   **Backend:** PHP, ASP.NET, Java Spring Boot, Node.js, Python (Django/Flask), Laravel
    *   **Database:** MySQL, PostgreSQL, SQL Server, Oracle, MongoDB (Relational structure/usage expected for the DB requirements)

---

## 2. Expected Features

Your application should incorporate most of the following features where applicable:

1.  User Registration and Login
2.  Role-Based Access Control (RBAC) (Admin/User/Specific Roles)
3.  User Profile Management
4.  Product/Service Management
5.  Shopping Basket / Shopping Cart
6.  Order Processing
7.  Inventory Management with Automatic Stock Updates
8.  Supply Chain / Order Tracking Module
9.  Interactive Dashboard with Charts and Reports
10. Blog / News / Announcement Module
11. Account Statistics and Summary Tables
12. Search, Filter, and Sorting capabilities
13. CRUD Operations (Create, Read, Update, Delete)
14. Report Generation (PDF / Excel / Print)
15. Data Validation and Error Handling

---

## 3. Deliverables

Each group must submit the following:

1.  **Software Requirement Specification (SRS)**

## 5. Important Instructions

*   **Unique Topics:** Every group must select a unique project topic.
*   **Software Engineering Practices:** Proper planning, design patterns, testing, and Git practices must be demonstrated.
*   **Plagiarism:** Copied code or plagiarism will result in disqualification.
*   **Vivas:** Every group member must contribute and be able to explain each module during the viva.
*   **Progress Reviews:** Regular progress reviews will be conducted by the project guide.
